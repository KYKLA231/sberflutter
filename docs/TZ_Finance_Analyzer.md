Проект: Finance Analyzer

Краткое описание
Приложение предназначено для удобного учета личных и семейных финансов. Основная цель — предоставить быстрый и понятный интерфейс для ввода доходов и расходов, анализа по категориям и валютам, а также простые отчёты и визуализации для принятия решений.

1. Общая архитектура

Платформа
 • Mobile (iOS / Android) — Flutter
 • Web (опционально)
 • Возможность локального хранения и (в будущем) облачной синхронизации

Тип пользователей
 • Пользователь (владелец кошелька)

Хранение данных
 • Локальная база (SharedPreferences для простых настроек, SQLite/Drift или Hive для транзакций и кэша)
 • Экспорт/импорт CSV/JSON
 • Автосохранение при добавлении/редактировании

2. Модель данных

Transaction (Транзакция)
 • Поля: id, title, amount, isIncome (bool), date, currency (ISO), category, notes

Category (Категория)
 • Предопределённые: Транспорт, Еда, Кафе/Рестораны, Магазины, Аптеки, Развлечения, Коммуналка, Другое
 • Возможность добавления/редактирования своих категорий

Currency (Валюта)
 • Поддержка нескольких валют (RUB, USD, EUR и т.д.)
 • Хранение валюты транзакции
 • (Опционально) Поддержка курсов конвертации для сводного баланса

3. Экран добавления транзакции

Поля
 • Название
 • Сумма
 • Валюта (Dropdown)
 • Категория (Dropdown)
 • Тип: Доход / Расход (чёткие кнопки)
 • Дата и время
 • Заметка (опционально)

Поведение
 • Валидация полей (сумма > 0, название не пустое)
 • Быстрое добавление (полноэкранная кнопка «Добавить»)
 • После добавления — возврат к анализу/списку и обновление данных

4. Отображение транзакций

Список
 • Хронологический список с фильтрами по дате/категории/валюте
 • Каждая строка: иконка категории, название, категория, дата, сумма с признаком +/- и цветом (зелёный/красный)

Детали транзакции
 • Просмотр и редактирование
 • Удаление с подтверждением

5. Баланс и аналитика

Баланс
 • Показать итоги по валютам (totalsByCurrency)
 • Опционально: сводный баланс в базовой валюте по курсу

Графики и отчёты
 • Диаграмма доходов/расходов (бар/пирог)
 • Фильтры: период (день, неделя, месяц, год, произвольно)
 • Топ категорий по расходам

6. Категории и товары/услуги

 • Возможность выбора категории при добавлении транзакции
 • Набор типовых категорий (Транспорт, Еда, Кафе/Рестораны, Магазины, Аптеки и т.д.)
 • Возможность пометить подкатегорию или тег (например: "продукты", "онлайн", "аптека")

7. Пользовательский поток (Welcome / Login)

 • При первом запуске — приветственный экран с вводом имени (сохранение в локальной базе)
 • Кнопка «Войти» переходит на главный экран
 • UX: приветствие "Добро пожаловать, [имя]"

8. Настройки

 • Валюта по умолчанию
 • Формат даты/locale
 • Экспорт/импорт данных
 • Очистка всех данных (с подтверждением и резервным экспортом)

9. Нефункциональные требования

 • Производительность: отклик UI < 100ms для основных операций
 • Offline-first: все функции базового учёта работают без сети
 • Лёгкость локализации (русский/английский)

10. Безопасность

 • Локальное хранение данных
 • (Опционально) Пин-код/биометрия для доступа к приложению
 • Резервное копирование при экспорте
